# informational-interview

Use when the user can talk to someone close to a target.

## Goal
Convert a vague chat into a deliberate evidence-gathering conversation.

## Inputs
- target
- current hypothesis
- 3-5 things the user most wants to verify
- who the conversation partner is

## Output
Produce:
- conversation goal
- 5-10 non-cringe questions
- for each question: what it is really testing
- red-flag answers
- green-flag answers
- what to infer indirectly instead of asking head-on
- post-call update checklist

## Rules
- do not optimize for sounding smart
- optimize for getting real texture
- prefer concrete past examples over abstract values talk
- avoid asking directly about dysfunction when it can be inferred indirectly

## Minimal recipe
1. state the hypothesis
2. list the evidence gaps
3. turn each gap into a question
4. define how answers should be interpreted
5. define what would change the current decision

## Good question shapes
- what did this role get pulled into unexpectedly in the last quarter
- what kind of person does unusually well here
- when this team is under pressure, what usually gives way first
- what does the manager tend to care about most in practice
- what would make someone leave this role after six months
